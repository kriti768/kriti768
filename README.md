## 💼 Featured Flagship Projects

### 1. [vLLM-Style Serving Engine & Scheduler](https://github.com/kriti768/systems-portfolio)
*A custom first-principles inference optimization serving layer implementing continuous iteration batching and paged memory allocations.*
* **Business Problem Solved**: Bypasses GPU memory reservation bottlenecks and execution queue starvation in traditional transformers.
* **Architecture**: Orchestrates requests in a unified event loop connecting a **Radix Prefix Cache** (LRU leaf node eviction) with a **Tree-Based Speculative Verifier** (width-2, depth-2 tree checked in a single pass using a specialized 7-token attention mask).
* **Tech Stack**: PyTorch, Python, FastAPI, Nginx (SSE stream routing), Prometheus Metrics.
* **Engineering Highlight**: Captured **1.6x - 1.8x throughput gains** and a **66% reduction in Time-To-First-Token (TTFT)** during benchmark loads.

### 2. [TinyCloud — Distributed Ingress API Gateway](https://github.com/kriti768/tinycloud)
*A high-throughput edge API gateway with dynamic routing, adaptive WAF rules, and decentralized rate limiting.*
* **Business Problem Solved**: Mitigates DDoS attacks and client overflow spikes at the entry point of microservice networks.
* **Architecture**: A Go reverse proxy cluster reading rules from a Redis cache, passing logs asynchronously through Apache Kafka telemetry pipelines to minimize proxy overhead.
* **Tech Stack**: Go, Redis, Apache Kafka, React, TypeScript.
* **Performance Highlight**: Successfully blocked **947 of 1000 concurrent attack requests** in load testing while sustaining 133 req/sec with ~98% proxy overhead reduction.

### 3. [CloudForge — Container Deployment Orchestrator](https://github.com/kriti768/CloudForge)
*A Vercel/Railway-inspired container orchestration platform for automated code-to-cloud deployments.*
* **Business Problem Solved**: Standardizes local developer deployments, secrets isolation, and system health observability.
* **Architecture**: Combines custom Docker Engine API integrations in Go with PostgreSQL for logs and state metadata, monitored via Prometheus & Grafana.
* **Tech Stack**: Go, Docker SDK, PostgreSQL, Redis, Apache Kafka, Prometheus, Grafana.
* **Key Features**: One-click rollback triggers, encrypted secrets storage, and live console build streams.

### 4. [AI Agent Sandboxed Execution Engine](https://github.com/kriti768/agent-sandbox)
*An event-driven sandbox system designed to compile and execute LLM-generated scripts securely.*
* **Business Problem Solved**: Executes untrusted code generated by AI agents without risking host system exposure.
* **Architecture**: Translates raw text queries into executable Python/JS packages via the Gemini API, running them inside resource-capped Docker containers.
* **Tech Stack**: Node.js, Express, Docker SDK, PostgreSQL, Gemini API, WebSockets.
* **Security & Performance**: Limits maximum CPU/Memory footprints dynamically, sustaining **193 req/sec with 100% successful sandbox launches**.
